# SECIモデルとAIエージェントの役割

## 1. SECIモデルの概要

SECIモデルは、知識がどのように創造され、組織内で共有・活用されるかを説明するフレームワーク。このモデルは、以下の4つの連続的なプロセスから構成される「知識創造サイクル」を提示する。

• 共同化 (Socialization) : 共通の体験を通じて、個人が持つ「暗黙知」（言語化しにくい経験的な知識）を他者に移転させるプロセス。
• 表出化 (Externalization) : 個人の暗黙知を、言葉や図などの「形式知」（言語化・共有可能な知識）に変換し、メンバーと共有するプロセス。
• 連結化 (Combination) : 既存の形式知と新たな形式知を組み合わせ、より体系的で新しい知識体系を創出するプロセス。
• 内面化 (Internalization) : 新たに得た形式知を、学習や実践を通じて個人の暗黙知として体得するプロセス。

## 2. SECIモデルにおけるAIエージェントの新たな役割

知識創造サイクルの各段階において、AIエージェントは以下のような専門的な役割を担う。

| 段階                        | エージェントの種類           | 具体的な機能                                                                                   |
| ------------------------- | ------------------- | ---------------------------------------------------------------------------------------- |
| **共同化 (Socialization)**   | **情報収集・構造化エージェント**  | <ul><li>従業員の会話・行動を収集</li><li>顧客の声をキャプチャ</li><li>マーケット情報を監視</li><li>暗黙知を構造化</li></ul>     |
| **表出化 (Externalization)** | **インサイト発見エージェント**   | <ul><li>情報を切片化</li><li>関連性をグルーピング</li><li>インサイトを生成</li><li>暗黙知を言語化</li></ul>             |
| **連結化 (Combination)**     | **戦略構築エージェント**      | <ul><li>実行可能な仮説を構築</li><li>施策・プロセスを設計</li><li>リスクベースのガバナンス</li><li>実験的アプローチを実装</li></ul> |
| **内面化 (Internalization)** | **実践支援・仮説検証エージェント** | <ul><li>実践現場での支援</li><li>市場での実験・検証</li><li>学習パスの最適化</li><li>新たな暗黙知の発見</li></ul>          |

## 所感
今やろうとしていることがこれのどこなのかを意識し、このサイクルのうち不足しているものがあれば、さらに展開してくようにしたい。一番よくやるのは、表出かなのだけど、その前提となる共同化や後続の内面化もやり魔将みたいなストーリーを立てたいところ。このモデルをまんま導入Phaseに落としてもいいかも

# AIエージェント導入における4つの壁
## 第1の壁：経営層の手触り感の不足

導入を推進する立場であるはずの経営層がそもそもAIを触っていない
* 手を動かす→概念化する→伝えるのプロセスが本来必要
* SB孫さんは2023年時点で毎日ChatGPTに触れていて、クリスタルのもととなる概念作り、SB内で実施した
* AIに触れてその実力や効果、限界などを体感していないにもかかわらず、現場に導入しろというのはナンセンス。経営層はまずは自分で触れてみて、実感を持って推進したい
	* →これがほとんどのエンタープライズでできていないのでは？現場には触りなさいと言っておきながら、自分は触らないのは違うよね？という話

## 第2の壁：組織への浸透の失敗

なかなか組織に浸透しないよねというもの。これはトップダウン-ボトムアップの両方のアプローチがいる

**トップダウンの役割**
3つの場を形成することが大事

| 環境設定の要素       | 具体的な施策                               |
| ------------- | ------------------------------------ |
| 実験を推奨する仕組み    | 予算の確保、短期的なROIを求めない、現場への権限移譲          |
| 知識共有のプラットフォーム | AIエージェントチャンピョンによる共有、ナレッジベース          |
| 心理的安全性の確保     | 「分からない」といえる文化の醸成、失敗を攻めない姿勢、調整を誉める雰囲気 |
**ボトムアップの役割**
とにかく実践と学習あるのみ。ただし、ROIを求めるよりも、業務が変わったという実績を作ることが大事
