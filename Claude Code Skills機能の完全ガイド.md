#AIエージェント 

Claude CodeのSkills機能は、**モジュラー化された専門知識をオンデマンドで提供する仕組み**である。2025年10月にAnthropicが正式発表したこの機能は、「一度作成すれば自動的に使用される」設計思想により、開発者の反復作業を大幅に削減する。Skillsはファイルシステム上のフォルダとして存在し、SKILL.mdファイルに記述された指示、スクリプト、リソースをClaudeが必要に応じて動的に読み込む。従来のCLAUDE.mdが「常にロードされる記憶」であるのに対し、Skillsは**タスクに関連する場合にのみ活性化する専門的な能力**として機能する。

## Skillsの定義と設計思想

Anthropicの公式ドキュメントによると、Agent Skills（Skills）は「エージェントが特定のタスクをより効果的に実行するために、動的に発見・読み込みできる命令・スクリプト・リソースの整理されたフォルダ群」と定義されている。この機能が生まれた背景には、汎用エージェントの能力向上に伴う新たな課題がある。

Anthropicのエンジニアリングブログは次のように説明する：「モデルの能力向上により、本格的なコンピューティング環境と対話する汎用エージェントを構築できるようになった。しかしこれらのエージェントがより強力になるにつれ、ドメイン固有の専門知識を装備するための、より**構成可能でスケーラブルかつポータブルな方法**が必要になった」。

Skillsの核となる設計目標は3つある。第一に**特化**—Claudeの能力を特定のドメインに合わせてカスタマイズすること。第二に**反復の削減**—同じガイダンスを複数の会話で繰り返し提供する必要をなくすこと。第三に**能力の組み合わせ**—複数のSkillsを組み合わせて複雑なワークフローを構築できること。

## Progressive Disclosureによる効率的なコンテキスト管理

Skillsシステムの技術的な特徴は、**3層の段階的開示（Progressive Disclosure）アーキテクチャ**にある。この設計により、必要な情報だけがコンテキストウィンドウに読み込まれ、トークン効率が最適化される。

**レベル1：メタデータ（約100トークン）** は起動時に常にロードされる。SKILL.mdのYAMLフロントマターに記述された`name`と`description`のみがシステムプロンプトに含まれ、Claudeはどのスキルがいつ使用されるべきかを判断できる。

**レベル2：インストラクション（5,000トークン未満）** はスキルがトリガーされた時にのみ読み込まれる。ClaudeがbashコマンドでSKILL.mdの本文を読み取り、初めてその内容がコンテキストに入る。

**レベル3：リソースとコード（実質無制限）** は参照された時にのみアクセスされる。追加のMarkdownファイル、スクリプト、アセットは必要に応じてClaudeがアクセスし、スクリプトの場合は実行結果のみがコンテキストに入る（コード自体は入らない）。

## 開発効率とチーム開発へのメリット

Skillsの導入による最大の利点は**コンテキスト効率の向上**である。従来、大規模なCLAUDE.mdファイルにすべての情報を詰め込む方式では、毎回のインタラクションで全トークンが消費されていた。Skillsはこの問題を解決し、関連するタスクでのみ専門知識がロードされる。

プロジェクトカスタマイズの観点では、Skillsの**ポータビリティ**が重要である。公式発表によると「Skillsはどこでも同じフォーマットを使用する。一度作成すれば、Claudeアプリ、Claude Code、APIで横断的に使用できる」。これにより、チームで作成したSkillsをプロジェクトの`.claude/skills/`ディレクトリにコミットすることで、全メンバーが同一の専門知識を共有できる。

チーム開発では以下のパターンが有効である：

- **プロジェクトSkills**（`.claude/skills/`）をGitリポジトリにコミットし、チーム全体で共有
- **個人Skills**（`~/.claude/skills/`）で実験的なワークフローを開発
- **プラグインSkills**を通じて組織全体に配布

## SKILL.mdの構造とフォーマット

すべてのSkillには`SKILL.md`ファイルが必須である。基本構造はYAMLフロントマターとMarkdown本文で構成される。

```markdown
---
name: my-skill-name
description: このSkillが何をするか、いつ使うべきかの簡潔な説明
---

# My Skill Name

## Instructions
Claudeが従うべき明確なステップバイステップのガイダンス

## Examples
具体的な使用例
```

フィールド要件は厳格に定められている。`name`は最大64文字で、小文字・数字・ハイフンのみ使用可能、「anthropic」「claude」などの予約語は禁止される。`description`は最大1,024文字で、XMLタグを含めることはできない。

オプションフィールドには`allowed-tools`（使用可能なツールの制限）、`license`（ライセンス情報）、`model`（使用モデルの指定）がある。

### ディレクトリ構造の標準パターン

```
skill-name/
├── SKILL.md              # コア命令（必須）
├── references/           # オンデマンドでロードされるドキュメント
│   └── *.md
├── scripts/              # 実行可能なPython/Bashスクリプト
│   └── *.py
└── assets/               # テンプレート、画像、ボイラープレート
    └── *
```

## 実践的なSkillファイルの例

### コードレビューSkill（ツール制限付き）

```yaml
---
name: code-reviewer
description: コードのベストプラクティスと潜在的な問題をレビューする。コードレビュー、PRチェック、コード品質分析時に使用。
allowed-tools: Read, Grep, Glob
---

# Code Reviewer

## レビューチェックリスト
1. コードの整理と構造
2. エラーハンドリング
3. パフォーマンスの考慮
4. セキュリティ懸念
5. テストカバレッジ

## 指示
1. Readツールでターゲットファイルを読み取る
2. Grepでパターンを検索
3. Globで関連ファイルを発見
4. コード品質に関する詳細なフィードバックを提供
```

## ユースケースの分類

コミュニティで確認された主要なユースケースは6つのカテゴリに分類される。

**開発・技術系**には、webapp-testing（Playwrightを使用したローカルWebアプリテスト）、test-driven-development（RED-GREEN-REFACTORサイクルの強制）、systematic-debugging（4フェーズの根本原因分析）が含まれる。

**ドキュメント処理系**では、docx/pdf/pptx/xlsxそれぞれの操作に特化したSkillsが提供されており、公式のプリビルトSkillsとしても利用可能である。

**クリエイティブ・デザイン系**には、algorithmic-art（p5.jsによるジェネレーティブアート）、frontend-design（プロダクショングレードのUI生成）などがある。

**エンタープライズ・コミュニケーション系**、**ワークフロー・プロセス系**、**メタSkills**（skill-creatorなど新しいSkillを作成するためのSkill）も活発に開発されている。

## CLAUDE.mdとの違いと連携

CLAUDE.mdとSkillsは補完的な関係にある。最も重要な違いは**ロードタイミング**である。

| 項目 | CLAUDE.md | Skills |
|------|-----------|--------|
| ロード | 起動時に常にロード | 関連タスク時にオンデマンド |
| トリガー | 自動（永続的コンテキスト） | モデル起動（タスクマッチング） |
| 目的 | プロジェクトの記憶と慣習 | 特化したタスク手順 |
| トークン消費 | 常に消費 | 活性化時のみ |

実践的な使い分けとして、CLAUDE.mdには「TypeScript strictモードを使用」「テストは`npm test`で実行」などの**常に適用されるべき規約**を記述する。一方、Skillsには「PDFレポートを作成して」と言われた時にのみ活性化する**専門的な手順**を記述する。

技術ブログの分析によると、「Skillsはモジュール化されたCLAUDE.mdの断片と考えることができる。Claudeが毎回巨大なドキュメントを確認する代わりに、必要な時にだけ特定の専門知識にアクセスできる」。

## ベストプラクティスと注意事項

Anthropicのエンジニアリングブログは4つの重要な実践を推奨している。

**評価から始める**：代表的なタスクでエージェントを実行し、苦戦する場所や追加コンテキストが必要な場所を観察して、その欠点に対応するSkillsを段階的に構築する。

**スケールを見据えた構造化**：SKILL.mdが扱いにくくなったら、内容を別ファイルに分割して参照する。相互排他的または一緒に使用されることが少ないコンテキストは、別々のパスに保持することでトークン使用量を削減できる。

**Claudeの視点で考える**：Claudeが実際のシナリオでSkillをどのように使用するかを監視し、観察に基づいて反復改善する。特にSkillのnameとdescriptionに注意を払う。

**Claudeと共に反復する**：タスクをClaudeと一緒に進める中で、成功したアプローチやよくある間違いを再利用可能なコンテキストとコードとしてSkillに取り込むようClaudeに依頼する。

### セキュリティに関する重要な注意

公式ドキュメントは明確に警告している：「Skillsは信頼できるソース—自分で作成したもの、またはAnthropicから入手したもの—のみを使用することを強く推奨する。Skillsは命令とコードを通じてClaudeに新しい能力を提供するが、これは強力である一方、悪意のあるSkillがツールを呼び出したりコードを実行したりする方法を、Skillの記載された目的と一致しない形で指示する可能性がある」。

## 結論

Claude Code Skillsは、AIエージェントに専門知識を装備するための次世代のアプローチを提示している。従来の「すべてをコンテキストに詰め込む」方式から「必要な時に必要な知識だけをロードする」パラダイムへの移行により、トークン効率と開発体験の両方が向上する。CLAUDE.mdとの適切な使い分け—永続的な規約はCLAUDE.mdに、タスク固有の専門手順はSkillsに—を理解することで、チーム全体で一貫した高品質な開発が実現できる。
